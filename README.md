# Channel Aggregator Telegram

## Для налаштування середовищя та встановлення бібліотек

1. Клонування проєкту

Щоб склонувати проєкт локально:
```bash
git clone git@github.com:Ascent-Group-Tech/Channel_Aggregator.git
```
Створення та перемикання на іншу(свою) гілку:
```bash
git checkout -b назва_вашого_модулю_гілки
```
Залити на синхронізувати вашу гілку з GitHub:
```bash
git push -u origin назва_вашого_модулю_гілки
```
Перевірка на які ви гілці зараз знаходитеся:
```bash
git branch
```
Додавання файлів до коміту:
```bash
git add .
```
> Крапка означає, додати всі зміненні файли до коміти. Але ви можете вказати і вручну

Коміт:
```bash
git commit -m "Ваш коментар щодо коміту"
```

Пушнути свою закомічені зміни на репозиторій Github:
```bash
git push
```

2. Налаштування середовищя:

Створити віртуальне середовищесередовище: 
```bash
python -m venv venv
```
Активувати його на Windows:
```bash
venv\Scripts\activate
```
Активувати на Linux:
```bash
source venv/bin/activate
```
І тільки після цього, Встановлення бібліотек проекту:
```bash
pip install -r requirements.txt
```
## Пояснення ключів та безпеки

`.env` файл який містить ключі для **Telegram** та глобальні данні не буде доступний в репозиторії, для безпеки(кращя практика)

Натомість є приклад `.env.example` який описує як має виглядати локальний `.env`, вносити ключі або інші важливі данні - **ЗАБОРОНЕНО**

файли `*.session` будуть теж виключені з репозиторію, лише локално, для безпеки сессій телеграм.


## Пояснення файлів

`config.py` завантажує з локального `.env` змінні які будуть доступні глобально у всьому проєкті

`main.py` - точка входу в програму

`core/client.py` - ядро та налаштування Юзер боту

`database/db_handler.py` - модуль роботи з базою данних

`logic/parser.py` - модуль парсеру тексту, виявлення лише валідних стрічок
`handlers/sync_handlers.py` - модуль для синхронізації/прослуховування телеграм каналів. PS: використовує інші модулі: бази данних, парсер

Розробити незалежну утілту модуль, для валідації стрічок(повідомлень).

2. ДатаБаза
Розробити модуль бази данних, який містить функції/методи для робити з базоданних, яка слугує для синхронізації id повідомлень між донрським каналом та головним.

3. Слухання телеграм каналу
Розробити модуль для прослуховування потрібного телеграм каналу використовуючи `pyrogram` бібліотеку.

**ВАЖЛИВО** - встановити випадкові затримки між запитами аби не отримати бан від Telegram за надмірне використання API.

Приклад функції для обробки подій:
```python
@Client.on_message(filters.chat("channel_id"))
async def handle_new_post(client, message):
    pass
```
Обов'язково навчиться використовувати декоратори та бібліотеку `pyrogram`. Ваш модуль має використовувати модулі `database/db_handler.py` та `logic/parser.py`
